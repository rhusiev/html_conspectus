# [[Шифр]] Цезаря
Замінюється кожна буква в алфавіті на букву, що стоїть на 3(якщо на ключ $k$ - то це [[Шифр]] зсуву) позицій правіше

Математично:
33 - кількість букв в українському алфавіті, в інших може бути інше(наприклад, 26 в англійському)
$\mathbf{Z}_{33}$ - $[0, 33)$ цілі числа
$p \in \mathbf{Z}_{33}: f(p) = (p + k)\ \mathbf{mod}\ 33$
$f^{-1}(p) = (p - k)\ \mathbf{mod}\ 33$

*Афінне перетворення* - $f(p) = ap + b\ \mathbf{mod}\ 33$
Але треба, щоб це була бієкція. Це бієкція, якщо $\gcd(a, 33) = 1$

Розшифрування:
$c = (ap + b)\ \mathbf{mod}\ 33$($\gcd(a, 33) = 1$). Треба виразити $p$ через $c$
$c \equiv ap + b \pmod{33}$
$c - b \equiv ap \pmod{33}$
$\gcd(a, 33) = 1 \Rightarrow \exists a^{-1}$
$a^{-1} (c - b) \equiv a^{-1}ap \pmod{33}$
$p \equiv a^{-1} (c - b) \pmod{33}$

Ці [[Шифр|шифри]] моноалфавітні, кожній букві відповідає одна буква
Можна просто пошуком ймовірностей використання букв в тексті відшукати букви, що відповідають буквам в [[Шифр|шифрі]]

# Блокові [[Шифр|шифри]]
**[[Шифр]] перестановки**
Ключ - перестановка $\sigma$ множини $\{1, 2, ..., m\}$, для якогось $m \in \N$ - *ін'єктивна* функція з $\l\{ 1, 2, ..., m \r\}$ у цю ж множину

Розбиваємо повідомлення на блоки довжиною $m$ букв(пунктуація, пробіли і т.д. не враховуються)
Якщо кінцевий блок менший за $m$, то доповнюємо його випадковими буквами

У нас є якась перестановка $\sigma$. Наприклад:
$m = 3, \sigma = \{ 0 \to 2, 2 \to 1, 1 \to 0 \}$
абввбаба -> абв вба баа -> бва бав ааб
$\sigma^{-1}$ - те, що зі зашифрованого дасть початкове

# Системи з відкритим ключем
Вони є *асиметричні* - є різні ключі для шифрування і розшифрування

## [[Шифр]] RSA
Відкритий ключ - пара $(n, e)$
Закритий ключ - число $d$

**Генерування**:
Пара простих $(p, q)$
$n = pq$ - *перше* число відкритого
$e$ - *друге*, якесь невелике непарне число, що взаємно просте з $(p - 1)(q - 1)$

$d \equiv e^{-1} \pmod{(p - 1)(q - 1)}$ - *закритий* ключ

**Шифрування**:
1. Повідомлення перетворюють у цифрову форму
Один зі способів:
А - 00, Б - 01,...Я - 32
Потім все в безперервну послідовність цифр
Розбивають на блоки довжиною $2N$ - таке максимальне, що $\underbrace{323232...32}_{2N} \le n$(щоб можна було робити однозначні операції за модулем $n$)
Якщо останній блок менший за $2N$, доповнюємо фіктивними
2. Кожен блок $M_i$ перетворюють за формулою $C = M^e \pmod{n}$
3. Розшифрування - $M = C^d \pmod{n}$


Виходить так, що тільки власник секретного ключа може прочитати

Щоб навпаки, тільки він міг зашифрувати, потрібно шифрувати через закритий $d$, а розшифровувати через відкритий $e$

Так можна робити, бо $\( x^e \)^d \mathbf{\ mod\ } n = \( x^d \)^e \mathbf{\ mod\ } n = x$

## Алгоритм обміну ключами
1. Аліса обирає велике просте $p$ та $r$ - первісний корінь за модулем $p$. Пересилає Бобові
2. Аліса та Боб вибирають секретні $k_1, k_2 \in (1, p-1]$
3. Кожен обчислює своє: $r^{k_1} mod p$ та $r^{k_2} mod p$, відкрито обмінюються
4. Кожен обчислює $r^{k_1k_2} mod p$ - це є секретний ключ
