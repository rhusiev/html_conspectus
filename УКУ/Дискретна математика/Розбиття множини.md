# Знаходження порядку елементів розбиття
Для A = {a, b, c} розбиття на k непорожні підмножини:
k = 1: {{a, b, c}}
k = 2: {{a, b}, {c}}, {{a, c}, {b}}, {{b, c}, {a}}
k = 3: {{a}, {b}, {c}}
$$\begin{flalign}
    &\Phi(n, k) - \text{кількість розбиттів на k непорожні підмножини, } n \ge 1&&\\
    &\Phi(n) - \text{кількість усіх розбиттів на непорожні підмножини}&&\\
    &\\
    &\Phi(n, k) - \text{числа Стірлінга другого роду}&&\\
    &\Phi(n) - \text{Числа Белла}&&\\
    &\\
    &\Phi(n, k) = \sum_{k=1}^{n}\Phi(n, k) &&\\
    &\Phi(n, k) = \Phi(n-1, k-1) + k\Phi(n-1, k) &&\\
    &\Phi(n+1)=\sum_{i=0}^{n} C_{n}^{i} \Phi(i) &&\\
\end{flalign}$$
## Перестановки
Для кожного A можна поставити у відповідність A'={1, 2, 3, ..., n}
Тобто ми впорядковуємо множину A, а далі будемо впорядковувати за A' розбиття

Кожну перестановку $(a_1, a_2, a_3...a_n)$ можна зобразити як $a_1a_2...a_n$

Тепер беремо лексикографічний порядок, визначаємо його, як:
$a_1a_2...a_{n} < b_1b_2...b_{n}$, якщо для якогось k, $1 \le k \le n$:
    $a_1=b_1,a_2=b_2,...,a_{k-1}=b_{k-1},a_{k}<b_{k}$

$b_1b_2...b_{n}$ лексикографічно наступна за $a_1a_2...a_{n}$, якщо $\not \exists c_1c_2...c_{n}$, що $a_1a_2...a_{n}<c_1c_2...c_{n}<b_1b_2...b_{n}$

Алґоритм побудови лексикографічного порядку:
- Ідемо від кінця, шукаємо перше місце, де $a_{i}<a_{i+1}$, напр. у 36*25*41 2 < 5
- У позицію i ставимо найменший елемент, що більший за $a_i$, тут мінімальне 1, але воно менше за 2, тому ставимо туди 4: 36*4*
- Переставляємо елементи, що залишилися після i в порядку зростання, тут 364*125*


## Сполучення
Тут теж ставимо у відповідність A'={1, 2, 3, ..., n}

Оскільки сполучення, то кожне розбиття з елементів якихось номерів те ж, що і цих номерів, але у впорядкованому порядку. Тобто {3, 5, 1} = {1, 3, 5}
У нашому випадку розглядаємо тільки впорядковані елементи({1, 3, 5}), щоб точно не повторюватися і було легше

Алґоритм побудови лексикографічного порядку:
- Ідемо від кінця, шукаємо, де можна збільшити число. Напр. у *123* з множини {1,2,3,4,5} можна 3 збільшити на один, наступне стане *124*
123 -> 124
- Якщо не можна, дивимося наступне, і збільшуємо його, а наступні є всі більші за нього в порядку зростання: *125* -> *134*. 132 не може бути, бо це те ж, що й 123, а воно раніше було
125 -> 134

# Генерування розбиттів множини
Ідемо поелементно, додаємо елементи до вже існуючих розбиттів
*Для першого елемента* розбиття поки {}, тому єдиний варіант є {{1}}
*Для другого елемента* розбиття монжа додати до вже існуючого елемента або створити новий:
    {{1}, {2}}, {{1, 2}}
*Для третього* робимо те ж, почергово для кожного з попередніх:
    Для {{1}, {2}} можна створити ще один елемент або додати до {1} або {2}:
        {{1}, {2}, {3}}, {{1, 3}, {2}}, {{1}, {2, 3}}
    Для {{1, 2}} додати до {1, 2} або створити новий елемент:
        {{1, 2}, {3}}, {{1, 2, 3}}
...
