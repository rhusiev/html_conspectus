The theory behind our algorithm

For this project, we're working with the transformer architecture. A transformer is a model, commonly used for natural language processing, that operates on sequences of data. In case of text, sub-word tokenization techniques, such as BPE, are usually used for representing a piece of text as a sequence of tokens belonging to a finitely sized dictionary. We will be focusing on decoder-only models and causal language modelling - the task of predicting the next token in a sequence.

Commonly, fine-tuning on task-specific data, aka transfer learning, is used to improve the transformer model's performance on a specific natural language processing task. This involves using a pretrained transformer model, which was trained on large amounts of data that's not specific to our task, and fine-tuning its parameters on a task-specific dataset to adapt it to the new task at hand. However, a drawback of this approach is that the number of parameters we need to tune is very large, leading to problems with training itself and with the deployment of trained models. During the fine-tuning process, all the model's parameters are updated based on the task-specific data. This requires computing gradients for each parameter, which can be computationally intensive, especially for large models. Moreover, the number of training iterations required for convergence can be substantial, further exacerbating the computational burden. Storing and loading the entire model into memory can be problematic as well, particularly on resource-constrained devices, which includes all consumer GPUs. Even on powerful machines, the memory requirements for fine-tuning can be prohibitive, limiting the practical applicability of these models in real-world settings. And deploying a full fine-tune means loading the full weights of the fine-tuned model - traditional fine-tuned models aren't at all modular, we might as well fully unload the original model when loading the fine-tuned variant. In the previous section, we discussed other methods commonly used to address the issue. We will be exploring LoRA - low rank adaptation - along with a few derivative works that attempt to tackle this issue.

LoRA

Researchers discovered that the embedding vectors are intrinsically low dimensional. Most of the data in them can usually be more efficiently expressed using a lower number of dimensions. The authors of the LoRA paper theorized that the attention matrices, therefore, should have intrinsically low rank. While we can't decompose these matrices, we can assume that during fine-tuning the change of the weights is also an intrinsically low rank matrix, and therefore limit it to be low rank from the get-go, by creating two matrices instead of one. Empirically, the authors discovered that only fine-tuning and this way allows us to get most of the quality that we'd normally get when fine-tuning all attention matrices, which is what we will focus on.

LoKR

Another approach is called LoKR. It's based on a technique called KronA, which uses the Kronecker product to go even further beyond low-rank assumptions. The idea behind LoKR is combining LoRA with KronA. As a result, with just as few trainable parameters, we can get a matrix with a significantly higher rank, while still not having the full rank (which is usually thought of as inefficient).

Implementation pipeline

All of these techniques discussed are generally presented as alternatives, none strictly better than the others. There are even techniques, such as Polytropon, that choose the best out of the fine-tuning techniques. Our goal is to implement a decoder-only transformer-based model, load an existing model's weights, implement these fine-tuning techniques for it and perform fine-tuning on a few tasks, comparing the fine-tuning results. So far we have gathered an understanding of a basic transformer's - such as GPT-2 - inner workings, and acquainted ourselves with the research that introduces these techniques. There are four main steps left: implementing a transformer model (we will use minGPT as a reference), planning out the implementation of the techniques, implementing the techniques themselves and fine-tuning on task-specific data with further benchmarking and reflection on the results.
